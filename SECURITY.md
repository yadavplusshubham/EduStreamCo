# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| latest `main` | ✅ |
| older releases | ❌ |

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Email your report to: **security@edustream.co.in**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

You will receive an acknowledgement within **48 hours** and a resolution timeline within **7 days**.

We follow [responsible disclosure](https://en.wikipedia.org/wiki/Responsible_disclosure) — once a fix is released, we will credit you (unless you prefer to remain anonymous).

## Scope

In scope:
- Authentication bypass
- Remote code execution
- Data exposure (API endpoints, MongoDB)
- XSS / injection vulnerabilities in the frontend or backend

Out of scope:
- Denial of service attacks
- Rate limiting issues
- Issues requiring physical access to the server
